4-Stage Pipeline

입력 이미지가 최종 Before/After 결과로 변환되기까지의 4단계 처리 과정입니다. 각 단계는 독립적으로 작동하며, 실패 시 다음 단계로 자동 폴백됩니다.

1

Face Detection

얼굴 인식 및 468개 랜드마크 탐지

MediaPipe Face Mesh를 사용하여 얼굴의 468개 3D 랜드마크를 실시간으로 탐지합니다. 이 중 입 주변 22개 랜드마크를 추출하여 치아가 보이는 정확한 영역을 계산합니다.

📍 핵심 랜드마크

  • 상부 내입술 11개: [78, 191, 80, 81, 82, 13, 312, 311, 310, 415, 308]
  • 하부 내입술 11개: [308, 324, 318, 402, 317, 14, 87, 178, 88, 95, 78]
  • 이 두 라인 사이 = 치아 노출 영역

⚙ 설정

  • maxNumFaces: 1
  • refineLandmarks: true (입/눈 정밀)
  • minDetectionConfidence: 0.5
  • CDN: @mediapipe/face_mesh@0.4
MediaPipe Face Mesh 468 Landmarks WASM
2

Teeth Segmentation

치아 영역 정밀 분리

Face Mesh 랜드마크로 생성한 폴리곤 내에서 실제 치아 픽셀만 분리합니다. 구강 내부(어두운 영역)와 잇몸(붉은 영역)을 제거하여 순수 치아 마스크를 생성합니다.

🎭 마스크 생성 과정

  • 내입술 랜드마크 → 폴리곤 클리핑
  • 구강 내부 제거 (밝기 < 80)
  • 잇몸 제거 (H < 0.06, S > 0.25)
  • 최종: 치아 픽셀만 남은 마스크

🔄 폴백: 색상 기반 감지

  • MediaPipe 실패 시 자동 전환
  • 입 영역 우선 감지 (입술 + 구강)
  • 상대 밝기 기준 치아 추출
  • Morphology 정리 (erosion + dilation)
// 치아 마스크 생성
const teethPoly = [
  ...upperInner.map(i => [landmarks[i].x * W, landmarks[i].y * H]),
  ...lowerInner.map(i => [landmarks[i].x * W, landmarks[i].y * H]),
];
ctx.beginPath();
ctx.moveTo(teethPoly[0][0], teethPoly[0][1]);
// → 폴리곤 클리핑 → 마스크 생성

3

AI Enhancement

Gemini AI 라미네이트 이미지 생성

Gemini 2.5 Flash Image 모델에 원본 사진과 스타일별 프롬프트를 전송합니다. AI가 치아만 변환한 새 이미지를 생성합니다. 3가지 스타일을 병렬로 동시 생성합니다.

3계층 폴백 구조
Gemini AI 이미지 생성 1순위 — 형태+색상+정렬 전부 변환
→
Canvas + MediaPipe 마스크 2순위 — 치아 영역 정밀 화이트닝
→
Canvas + 색상 감지 3순위 — 자동 감지 기반 처리

🚀 API 호출

  • 모델: gemini-2.5-flash-image
  • 입력: 원본 이미지 + 프롬프트
  • 출력: 수정된 이미지 (base64)
  • 3가지 스타일 병렬 생성 (Promise.allSettled)

🎨 스타일별 VITA shade

  • 자연형: A2 (확실히 밝지만 자연스러움)
  • 프리미엄형: A1 (고급 밝기 + 포슬린 질감)
  • 화이트형: B1/Bleach (최대 밝기 순백)

4

Result & Storage

결과 출력 + Supabase 영구 저장

생성된 Before/After 이미지를 슬라이드 비교 UI로 출력하고, 6가지 항목 AI 분석 점수와 함께 Supabase DB에 영구 저장합니다.

📊 출력

  • Before/After 슬라이드 비교
  • 나란히 보기 (Side-by-side)
  • 3가지 스타일 탭 전환
  • 6항목 점수 + 등급(S~D) + 추천 치료
  • Before/After 이미지 다운로드

🗄 저장 (Supabase)

  • patients: 고객 정보
  • simulations: 점수 + 이미지(base64)
  • Before + After 3종 = 이미지 4장 저장
  • 환자 관리에서 이력 조회 가능
